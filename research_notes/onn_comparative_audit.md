# Operation Neural Network (ONN) Architecture Audit

> Note: This is a research note. It may discuss methods that are not implemented in the current Glassbox codebase.

We compare ONN to related methods, analyze its internal dynamics, and propose fixes. Our focus is on convergence stability, interpretability, and speed.

## 1\. Comparative Analysis

### ONN vs DARTS (Differentiable NAS)

*   **Continuous relaxations and discretization collapse:** Like DARTS, ONN uses continuous architectural parameters (Hard Concrete gates) that must be “snapped” to discrete choices. In DARTS this often causes a **performance collapse**: a well-performing soft supernet can yield a poor discrete network[1](https://arxiv.org/html/2405.16610v1#:~:text=practical%20applications%20of%20DNAS%20are,supernet%20might%20underperform%20after%20retraining). Chen _et al._ describe this as a “discretization error”: if many operations have nonzero weight (high entropy), then pruning them (to one-hot) can drastically change the model[1](https://arxiv.org/html/2405.16610v1#:~:text=practical%20applications%20of%20DNAS%20are,supernet%20might%20underperform%20after%20retraining). In ONN we observe exactly this when rounding the learned exponent $p\approx1.97$ to $2.0$ causes error spikes – analogous to DARTS’s entropy gap.
*   **Entropy regularization:** DARTS solutions include enforcing one-hot (low-entropy) distributions on architectural weights. For example, FairDARTS replaces the softmax with independent sigmoids (removing skip-connection “unfair advantage”)[2](https://www.ecva.net/papers/eccv_2020/papers_ECCV/papers/123600460.pdf#:~:text=solution%20back%20to%20a%20discrete,drives%20skip%20connections%20into%20a). Other works add an entropy penalty or Gumbel relaxation to bias discrete selection[3](https://arxiv.org/html/2405.16610v1#:~:text=One%20of%20the%20common%20approaches,We). ONN can similarly anneal entropy: gradually strengthen the gate regularizer (or raise Hard Concrete temperature) to push each operation choice toward 0/1. However, caution is needed: Chen _et al._ show that too-strong entropy loss in DARTS trades off searching performance for discretization[3](https://arxiv.org/html/2405.16610v1#:~:text=One%20of%20the%20common%20approaches,We). In practice, an **annealing schedule** (weaker penalty early, stronger later) is common.
*   **Skip-connection bias / “unfair advantage”:** DARTS famously suffers when skip-connections dominate the architecture (they give an “unfair advantage” during competition), causing collapse[2](https://www.ecva.net/papers/eccv_2020/papers_ECCV/papers/123600460.pdf#:~:text=solution%20back%20to%20a%20discrete,drives%20skip%20connections%20into%20a). FairDARTS addresses this by decoupling operations so no operation can saturate others[4](https://www.ecva.net/papers/eccv_2020/papers_ECCV/papers/123600460.pdf#:~:text=monopoly%20state%20in%20exclusive%20competition,first%20collaborative%20competition%20approach%20by). In ONN a similar imbalance could arise if certain meta-ops (e.g. identity or linear combos) consistently win during search. Ensuring _fair_ competition – for example by clamping gate outputs or using independent sigmoid gates – can prevent a single op from monopolizing the formula, as FairDARTS achieves in CNN search[2](https://www.ecva.net/papers/eccv_2020/papers_ECCV/papers/123600460.pdf#:~:text=solution%20back%20to%20a%20discrete,drives%20skip%20connections%20into%20a).
*   **Search vs Structure:** DARTS searches for network cells (with conv, skip, etc.) via gradient descent. ONN instead searches a _mathematical expression DAG_. The analogy is that ONN’s meta-operations (MetaPower, MetaTrig, etc.) are like the operation choices in DARTS cells. Both suffer from a huge search space and continuous→discrete gap. However, unlike DARTS which has fixed topology and only chooses among operations on edges, ONN must also select input wires (routing) on a fully flexible DAG. This larger search space exacerbates instability. Techniques from DARTS (e.g. Gumbel-Softmax for discrete sampling, straight-through estimators[5](http://bayesiandeeplearning.org/2016/papers/BDL_8.pdf#:~:text=MPI%20Tubingen%20%C2%A8%20sg717%40cam,Poole%E2%88%97)) directly carry over, but may need reinforcement learning style variance-reduction if gating variance is high.

### ONN vs KAN

*   **Basis Functions – Local vs Global:** Traditional KANs use _learnable B-spline basis functions_ on each edge[6](https://arxiv.org/html/2510.25781v3#:~:text=The%20original%20KAN%20architecture%20employs,architectural%20variants%2C%20optimization%20strategies%2C%20and). B-splines have compact support (local control) and continuity properties, so each input sample affects only nearby spline segments[6](https://arxiv.org/html/2510.25781v3#:~:text=The%20original%20KAN%20architecture%20employs,architectural%20variants%2C%20optimization%20strategies%2C%20and). ONN’s meta-ops (MetaPower, MetaArithmetic, MetaTrigonometric, etc.) are _global_ functions: they apply across the entire input domain. Local bases can approximate arbitrary functions flexibly, but require many segments for sharp features; global ops represent entire functions (e.g. exact $\sin$ curve) but may be harder to fit gradually.
*   **Complexity and Computation:** B-splines offer smooth piecewise polynomials, but their evaluation and gradient can be slow if many knots are used[8](https://ar5iv.labs.arxiv.org/html/2509.10089v1#:~:text=functions%20are%20represented%20using%20B,Additionally%20the%20evaluation). Indeed, KAN-SR replaced B-splines with radial basis functions for efficiency[7](https://ar5iv.labs.arxiv.org/html/2509.10089v1#:~:text=of%20the%20splines%20is%20slow,basis%20functions%20for%20greater%20efficiency). ONN’s global ops (power, exp, etc.) are simpler per evaluation, but could still overflow/underflow. In terms of **interpretability**, a B-spline KAN yields learned curve shapes that are less obviously “nice” functions, whereas ONN’s end goal is a closed symbolic expression. KAN literature emphasizes that _basis choice governs interpretability_: certain bases (e.g. polynomials) produce smoother, more global features, while others (e.g. B-splines) give locality[6](https://arxiv.org/html/2510.25781v3#:~:text=The%20original%20KAN%20architecture%20employs,architectural%20variants%2C%20optimization%20strategies%2C%20and). ONN fixes the basis (meta-ops) by design, trading this adaptivity for a predefined functional vocabulary.
*   **Stability and Training:** KANs (with appropriate basis) often converge reliably and avoid the combinatorial pitfalls of symbolic search, at the cost of potentially slower per-iteration time. ONN’s global operations can lead to optimization plateaus (see MetaArithmetic below) or explosion (e.g. MetaExp). KAN research finds that learning inner functions with B-splines can require tuning (knot placement, regularization)[8](https://ar5iv.labs.arxiv.org/html/2509.10089v1#:~:text=functions%20are%20represented%20using%20B,Additionally%20the%20evaluation). ONN doesn’t need that, but the flip side is the discrete snap difficulty. In summary, KAN’s local bases might yield more gradual learning curves (no hard rounding needed), while ONN’s global ops can represent exact formulas if trained well, but are brittle under discretization.

## 2\. Gradient Stability & Routing Diagnosis

### MetaArithmetic blending function

ONN’s _MetaArithmetic_ node blends addition and multiplication via a parameter $\beta$: $$f_\beta(x,y) = (2-\beta)(x+y) + (\beta-1)\,(x\,y)\,. $$ This linearly interpolates between $(x+y)$ at $\beta=1$ and $(x\,y)$ at $\beta=2$. The gradient of $f$ w.r.t. $\beta$ is $$\frac{\partial f}{\partial \beta} = - (x+y) + (x\,y) = x\,y - (x+y)\,. $$ Critically, this is _zero_ whenever $x y = x+y$ (e.g. $x=y=2$ or more generally $(x-1)(y-1)=1$). At those points $\beta$ is a stationary parameter – small changes in $\beta$ do not change $f$. In practice, this means if the data contains points where $xy\approx x+y$, the learning signal for $\beta$ can vanish, creating **plateaus** in the loss landscape. Near such input regions, $\beta$ sees little gradient so the model can get “stuck” at $\beta\approx1.5$ (the midpoint). This introduces a saddle-like flat region: on one side, $x+y$ dominates ($\beta$ lower), on the other $x\,y$ dominates ($\beta$ higher), but near the boundary the model has little preference. Empirically, this can slow or destabilize convergence of $\beta$. A remedy could be to reparameterize the blend (e.g. a sigmoid interpolation) or to scale $x,y$ to avoid frequent $(x-1)(y-1)\approx1$ cases.

### Input Routing and Gumbel-Softmax

ONN may route multiple inputs into a MetaOp node using a categorical choice (e.g. picking one of several predecessor nodes). If Gumbel-Softmax or Hard Concrete is used for this discrete routing, it can introduce variance. In general, Gumbel-Softmax relaxations provide **low-variance, pathwise gradients**, and have been shown to **outperform** score-function (REINFORCE) estimators in similar settings[9](http://bayesiandeeplearning.org/2016/papers/BDL_8.pdf#:~:text=can%20approx%02imate%20categorical%20samples%2C%20and,costly%20marginalization%20over%20unobserved%20categorical). The _Straight-Through Gumbel_ variant (discretize forward, use continuous backward) is common: it lets forward passes use one-hot samples while backpropagating through the soft sample[5](http://bayesiandeeplearning.org/2016/papers/BDL_8.pdf#:~:text=MPI%20Tubingen%20%C2%A8%20sg717%40cam,Poole%E2%88%97). This yields a low-bias, low-variance update that often converges faster than REINFORCE. Indeed, Jang _et al._ demonstrate that Gumbel-Softmax “outperforms all single-sample gradient estimators” for categorical latent variables[9](http://bayesiandeeplearning.org/2016/papers/BDL_8.pdf#:~:text=can%20approx%02imate%20categorical%20samples%2C%20and,costly%20marginalization%20over%20unobserved%20categorical).

For ONN, if routing variance is problematic, we recommend: - **Straight-Through (ST) estimator:** use hard one-hot selection in the forward pass but pretend it was the soft sample during backprop[5](http://bayesiandeeplearning.org/2016/papers/BDL_8.pdf#:~:text=MPI%20Tubingen%20%C2%A8%20sg717%40cam,Poole%E2%88%97). This has been effective in many neural NAS methods. - **Rao-Blackwellized or variance-reduced estimators:** as noted in generative models, one can average over Gumbel noise to cut variance. If necessary, Reinforce with a learned baseline could also be applied, though its variance is typically higher[9](http://bayesiandeeplearning.org/2016/papers/BDL_8.pdf#:~:text=can%20approx%02imate%20categorical%20samples%2C%20and,costly%20marginalization%20over%20unobserved%20categorical). - **Entropy or confidence regularization:** encourage the routing distribution to be peaky (low entropy) only near convergence. Annealing the Gumbel temperature or adding an entropy penalty (weighted over epochs) can make routing decisions more stable.

## 3\. Search Space Optimization and Regularization

### Sparsity and Parsimony via Regularization

*   **Hoyer regularization:** The Hoyer measure (ratio of L1 to L2 norms) is a scale-invariant sparsity metric. Regularizing toward higher Hoyer sparsity has been shown to induce _much sparser_ activations (or weights) than an L1 penalty alone[10](http://proceedings.mlr.press/v119/kurtz20a/kurtz20a.pdf#:~:text=that%20sparsity%20can%20be%20significantly,simplifies%20the%20fine%02tuning%20process%2C%20since). Kurtz _et al._ find that combining Hoyer-regularization with thresholding yields high sparsity “with no accuracy loss”[10](http://proceedings.mlr.press/v119/kurtz20a/kurtz20a.pdf#:~:text=that%20sparsity%20can%20be%20significantly,simplifies%20the%20fine%02tuning%20process%2C%20since). In ONN, one could apply a Hoyer penalty to the vector of active MetaOp coefficients (or powers $p$) to push as many as possible to zero/one. This encourages a parsimonious formula (few operations with non-negligible contribution). Practically, adding $\lambda_H (|w|_1 / |w|_2)$ or its smooth approximation to the loss will shrink excess terms.
*   **Gate (L0) regularization:** Using Hard Concrete gates naturally implements a kind of $L_0$ regularizer: Louizos _et al._ show that with their hard-concrete distribution, the expected number of active gates is differentiable w.r.t. gate parameters[11](https://arxiv.org/abs/1712.01312#:~:text=zero,We). In practice, ONN can explicitly add an $L_0$-like penalty on gate probabilities (e.g. pushing each gate $\alpha$ to favor 0 or 1) to penalize extra branches. This parallels how Neural Architecture Search methods incorporate cost on connections. Such a penalty will encourage the model to drop unnecessary ops. Empirical SR methods often reward simplicity (e.g. with MDL/BIC)[12](https://arxiv.org/html/2406.06751v2#:~:text=Reward%20Design,to%20the%20principle%20of%20minimum); an $L_0$ gate regularizer achieves the same effect continuously.
*   **Combined complexity penalty:** In addition to sparsity of gates, one can directly penalize the symbolic complexity. For example, adding a _BIC-inspired_ term (proportional to number of parameters/operators) to the loss was shown to balance accuracy vs equation length[12](https://arxiv.org/html/2406.06751v2#:~:text=Reward%20Design,to%20the%20principle%20of%20minimum). Specifically, a Bayesian information criterion or minimum-description-length penalty will “automatically adjust the trade-off between expression complexity and data fit”[12](https://arxiv.org/html/2406.06751v2#:~:text=Reward%20Design,to%20the%20principle%20of%20minimum). In ONN this might mean adding $c \cdot #{\text{active ops}}$ or $c \cdot \sum |w_i|^0$ to the loss, with $c$ tuned so that performance does not degrade.

### Pruning Meta-Operations

Monitor which meta-operations consistently fail to contribute. For instance, if **MetaExp** often produces extremely large values or dead neurons (a common failure), consider removing it from the library. Similarly, if **MetaAggregation** (if present) is never chosen by gates, drop it to shrink the search space. This is analogous to “operation dropout” ideas in NAS (e.g. DropNAS) where seldom-used ops are dropped. Pruning can be automated: after a few search runs, identify ops with near-zero gate probability across trials and exclude them in future runs. This reduces variance and search time. Note that each meta-op introduces its own numerical risks (e.g. $\exp(x)$ can overflow, $x^p$ with large $p$ can vanish for $|x|<1$), so pruning unstable ops often aids convergence.

## 4\. Proposed Solutions

### Tier 1 (Engineering Fixes)

*   **Complexity-penalized loss:** Add an explicit term to the loss that penalizes formula length or operation count (e.g. a BIC-like penalty[12](https://arxiv.org/html/2406.06751v2#:~:text=Reward%20Design,to%20the%20principle%20of%20minimum)). This guides the search toward simpler expressions and can prevent overly complicated soft solutions that fail when discretized.
*   **Entropy annealing:** Start with mild entropy/gumbel penalties and gradually increase them. Early on, allow exploration with multi-op combinations; later, force decisions by penalizing entropy (as in [3](https://arxiv.org/html/2405.16610v1#:~:text=One%20of%20the%20common%20approaches,We)). This mirrors curriculum learning for architecture.
*   **Smooth discretization (“soft rounding”):** Instead of a hard switch $p\to2.0$, one can continue using the near-integer exponent for inference (e.g. keep $p=1.97$ if it fits better) and only gradually nudge to the integer. Concretely, perform _progressive rounding_: replace $p$ by a differentiable surrogate that has a plateau around integers. For example, use $p' = \text{round}_\tau(p)$ where $\tau$ controls smoothness. This can help the network adapt to the ultimate discrete setting smoothly, rather than an abrupt snap that causes collapse.

### Tier 2 (Hybrid Evolutionary/Gradient)

*   **Lamarckian weight inheritance:** When evolving DAG structures, pass the optimized continuous weights (e.g. exponent $p$, coefficients of operations) from parent to child solutions. Prellberg and Kramer show that such Lamarckian evolution dramatically speeds NAS convergence by reusing learned weights[13](https://arxiv.org/abs/1806.08099#:~:text=,in%20data%20efficiency%20is%20observed). In ONN, if an ES mutation only slightly changes structure, inheriting the previous continuous parameters (and then fine-tuning) can save much computation. This hybrid of evolution + gradient leverages the best of both: structure innovation via ES, parameter tuning via gradient.
*   **Gradient-informed mutations:** Use $\partial \mathcal{L}/\partial \alpha$ (the architecture gradient) to guide which mutations to apply. For example, if the gradient wrt a particular operation’s weight is near zero, that op might be pruned or replaced. This is akin to “neuron saliency” methods. One could compute sensitivity of loss to each operation (e.g. absolute gate gradient) and bias mutations (crossover or dropout) toward less sensitive parts of the DAG. This focuses search on the most promising sub-graph, reducing wasted exploration.
*   **Hybrid search:** Combine evolutionary branching with occasional gradient updates of meta-parameters. For instance, alternate ES steps (structure mutation) with short gradient descent bursts on continuous params. Each new structure uses the previous weights as initialization (Lamarckian) and refines them. This cyclical approach often yields faster convergence than pure ES or pure GD.

### Tier 3 (Radical Redesign)

*   **Transformer or autoregressive generators:** Instead of a fixed DAG with local gates, use a sequence or tree-generation model (e.g. a transformer or RNN) to _generate_ whole expressions token-by-token. Recent works use pretrained Transformers to output symbolic expressions from data[14](https://proceedings.neurips.cc/paper_files/paper/2023/hash/8ffb4e3118280a66b192b6f06e0e2596-Abstract-Conference.html#:~:text=Symbolic%20regression%20,based%20Planning%20strategy%20for). One could train a Transformer (or diffusion model) on pairs (data → equation) and use it to propose candidate formulas. This bypasses gradient gating entirely and leverages deep language-modeling advances. If combined with a planner or MCTS (as in TPSR[14](https://proceedings.neurips.cc/paper_files/paper/2023/hash/8ffb4e3118280a66b192b6f06e0e2596-Abstract-Conference.html#:~:text=Symbolic%20regression%20,based%20Planning%20strategy%20for)), it can incorporate accuracy/complexity feedback. The downside is training such generative models from scratch can be costly.
*   **Mask-generated sparse expression trees:** Use a single neural “template” (e.g. a full binary tree of a fixed depth) and learn a binary mask over edges/nodes to prune it into a valid expression. Instead of gating each edge softly, maintain a latent mask vector (or sparse attention) that is learned jointly with tree parameters. Techniques like differentiable top-k or continuous relaxation of subset selection (e.g. sparsemax or concrete subset) could produce sparse selection without Gumbel noise. Alternatively, formulate the search as a constraint satisfaction: each valid expression is a point in a huge discrete space, which one could tackle with a learned SAT/ILP solver or combinatorial optimization module. For example, use a pointer network to construct the tree while enforcing syntactic rules. These approaches are more complex but eliminate the pitfalls of continuous relaxations.
*   **Alternative architecture search methods:** If Hard Concrete proves problematic, one might replace it with other discrete selection frameworks. For example, use **neural SAT solvers** or **Gumbel-Matching** formulations to enforce exactly one parent per gate. Or employ **reinforcement learning** policies (REINFORCE with sophisticated baselines) to choose operations, trading off gradient bias for potentially lower variance or escape from saddle points. In essence, consider treating the final discrete symbol search as a black-box combinatorial problem guided by learned heuristics, rather than pure gradient descent.
