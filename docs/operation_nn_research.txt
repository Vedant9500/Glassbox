# Operation-Based Neural Network: Research Document

> **Project:** Glassbox ONN  
> **Last Updated:** 2026-01-21

---

## Vision Statement

Design a **new type of neural network** where:
- **Edges** carry VALUES (constants like 0.5, 1, 2, 3)
- **Neurons** ARE OPERATIONS (sin, +, *, ^2, log, etc.)
- **Multi-input aggregation**: [1,2,3] → + neuron → 6

This is a fundamental architectural redesign, not just symbolic regression.

---

## Architecture Diagram

```
TRADITIONAL NN:
    input ──[w1]──> (σ) ──[w2]──> output
              └─ weighted sum + activation

OUR ONN:
    [1] ────┐
            ├──> ( + ) ──> [6]
    [2] ────┤
            │
    [3] ────┘
    
    Edge = VALUE    Neuron = OPERATION
```

## Key Research Findings

### 1. The "EmbedGrad-Evolution" Framework

From `docs/research.md`:

**Hybrid optimization combining:**
- **Meta-Operations**: Continuous parametric ops (replaces discrete selection)
- **Differentiable Routing**: Learn which inputs connect to which slots
- **Evolution**: Global topology search
- **L-BFGS**: Local constant fitting

### 2. Meta-Operations (Core Innovation)

Replace 29 discrete operations with 4 parametric families:

| Meta-Op | Formula | Recovers |
|---------|---------|----------|
| `MetaPeriodic` | `A·sin(ωx + φ)` | sin, cos, linear |
| `MetaPower` | `sign(x)·\|x\|^p` | identity, square, sqrt, inv |
| `MetaArithmetic` | `(2-β)(x+y) + (β-1)(xy)` | add, multiply |
| `MetaAggregation` | softmax weighted | max, min, mean |

**Key insight**: This converts discrete search → continuous optimization

### 3. Critical Technical Solutions

| Problem | Solution |
|---------|----------|
| Discrete choices not differentiable | **Meta-Operations** (continuous) |
| Gumbel-Softmax never exact | **Hard Concrete Distribution** |
| Variable arity | **Differentiable Routing Matrix** |
| Gradient variance between ops | **BatchNorm after each op** |
| Local minima | **Hybrid Evo + Gradient** |
| RNN sequential bias | **DAG Architecture** |

### 4. Loss Function Design

```
L = L_mse + λ1·||c||_1 + λ2·ΣH(p_i) + λ3·Complexity
    ↓         ↓              ↓            ↓
  fit data  sparsity   force discrete  parsimony
```
